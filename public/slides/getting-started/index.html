Getting Started

ConFigTree

From Config.xlsx to a typed C# class in UiPath Studio

configtree.cprima.net

What you need

  • A REFramework project in UiPath Studio (2023.10.12 or later)
  • Your project's Config.xlsx
  • UiPath.CodedWorkflows package on your feed

The 8 steps

  1. Generate the class & snippet
  2. Install UiPath.CodedWorkflows
  3. Download Config.cs
  4. Import the namespace
  5. Paste the XAML snippet
  6. Convert variable → argument
  7. Wire up in Main.xaml
  8. Verify Project

1Generate the class & snippet

Open configtree.cprima.net and drop Config.xlsx onto the drop zone.

C# class tab

Scan the generated class — confirm namespace, class name, and properties.

XAML snippet tab

Optionally change Variable name in the sidebar, then click Regenerate. Click Copy.

2Install UiPath.CodedWorkflows

In Studio: Manage Packages → search UiPath.CodedWorkflows → Install.

This enables .cs source files inside a Studio project.
Tested with Studio 2023.10.12 and later.

3Download Config.cs

Switch to the C# class tab → click Download.
Save as Config.cs into your project's Lib/ folder.

Studio picks up .cs files from anywhere inside the project folder — Lib/ is a convention, not a requirement.

4Import the namespace

Open Framework/InitAllSettings.xaml → Imports panel → add:

Cpmf.Config

This also pulls in the assembly reference. Studio resolves CodedConfig from Config.cs.

5Paste the XAML snippet

In InitAllSettings.xaml, scroll to the bottom of the sequence, click after the last activity, press Ctrl+V.

The pasted block includes:

  • A ForEach reading each config sheet into a Dictionary<string, DataTable>
  • An Assign calling CodedConfig.Load(dt_Tables)
  • (if assets) One GetRobotAsset per asset property

6Convert variable → argument

Variables panel → right-click out_ConFigTree → Convert to Argument.

Studio gets two things wrong — fix both:

  • Direction: In → change to Out
  • Type: Object → change to CodedConfig (from Cpmf.Config)

7Wire up in Main.xaml

  1. Right-click InitAllSettings.xaml → Find References
  2. Open Main.xaml → create variable v_ConFigTree of type CodedConfig
  3. Import Cpmf.Config in Main.xaml's Imports panel
  4. On the InvokeWorkflowFile → Import Arguments → map out_ConFigTree → v_ConFigTree
  5. Add In argument in_ConFigTree : CodedConfig to Process.xaml and map it

8Verify Project

Press Ctrl+Shift+E — a clean result confirms the integration is complete.

Errors? See the Troubleshooting page in the wiki.

Using the typed config

Before — string key lookup

in_Config("Settings_QueueName")

After — typed, IntelliSense

in_ConFigTree.Settings.QueueName

Both work in the same workflow during gradual migration.
